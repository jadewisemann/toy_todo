# 개발 과정 문서

## Day 2 - 2026-05-13

### 목표

로그인 흐름을 최소 기능 범위로 도입한다. 인증 기능은 이후 Todo API 변경의 기반이 되므로, 먼저 페이지 구조와 보호 라우팅을 정리한다.

### 구현 범위

#### 인증

- 인증 방식: 세션 기반 인증
- 회원가입
- 로그인
- 로그아웃
- 사용자 입력 필드
  - 아이디
  - 비밀번호
  - 이름
- 클라이언트는 세션 존재 여부를 기준으로 인증 상태를 판단한다.
- 인증 상태 확인은 서버 API 응답을 기준으로 한다.

#### 라우팅

- 라우터 도입
  - `react-router` 사용
- 페이지 3개 구성
  - 로그인 페이지
  - 회원가입 페이지
  - Todo 페이지
- 보호 라우트 추가
  - 인증되지 않은 사용자는 Todo 페이지 접근 제한
  - 인증된 사용자는 로그인/회원가입 페이지 접근 흐름 정리

#### 내비게이션

- Nav bar 추가
- 인증 상태에 따른 조건부 렌더링
  - 비로그인 상태: 로그인, 회원가입 진입 표시
  - 로그인 상태: 사용자 정보, 로그아웃 표시

### 구조 정리 방향

인증 관련 컴포넌트와 훅은 `features/auth` 기준으로 묶는 방향을 우선 검토한다.

예상 구조:

```text
src/
  app/
    router.tsx
  features/
    auth/
      components/
      hooks/
      pages/
    todos/
      components/
      hooks/
      pages/
  shared/
    components/
```

라우팅, 전역 레이아웃, Todo 기능과 인증 기능의 책임이 섞이지 않도록 분리한다.
라우터 설정은 `app/router.tsx`에서 관리하고, 라우팅 대상 페이지는 각 feature 내부의 `pages`에 둔다.

### 작업 순서

1. 라우터와 페이지 구조를 먼저 만든다.
2. 인증 페이지의 최소 입력 폼을 만든다.
3. 인증 상태를 관리할 위치를 정한다.
4. 세션 기반 로그인, 회원가입, 로그아웃 흐름을 연결한다.
5. Nav bar 조건부 렌더링을 적용한다.
6. 보호 라우트를 추가한다.
7. Todo API 변경 범위를 확인한다.

### 보류 항목

- Todo API 변경
  - 인증 도입 이후 사용자별 Todo 데이터 구조가 필요한지 확인한다.
  - 세션 기반 인증에서 Todo API 요청/응답 흐름을 어떻게 처리할지 결정한다.
